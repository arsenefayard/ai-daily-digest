Résumé complet du projet — AI Daily Digest
Date de rédaction : 11/03/2026

1) Objectif global du projet
- Construire un "daily digest" multi-thèmes (IA, géopolitique, économie, sport, musique, science, culture, histoire) consultable en format carte mobile-first.
- Générer automatiquement les contenus via Perplexity.
- Publier les pages statiques sur GitHub Pages (branche gh-pages).
- Envoyer un email quotidien avec accès au digest.
- Ajouter une fonction interactive "loupe" sur chaque carte pour poser des questions sur le contenu.

2) Architecture générale
- Frontend statique : 8 fichiers HTML
  - index.html (IA)
  - geo.html
  - eco.html
  - sport.html
  - music.html
  - science.html
  - culture.html
  - history.html
- Génération backend : scripts Python
  - ai_digest.py, geo_digest.py, eco_digest.py, sport_digest.py, music_digest.py, science_digest.py, culture_digest.py, history_digest.py
- Workflow CI/CD : .github/workflows/daily-digest.yml
- Données publiées : today.json, history.json et variantes par thème sur gh-pages.

3) Demandes utilisateur initiales
- Corriger l'affichage en mode paysage mobile (scroll vertical impossible sur certaines cartes).
- Éviter les textes tronqués qui cassent la compréhension.
- Pour la section IA : prioriser les vraies avancées (nouveaux modèles, percées, lancements concrets), pas les news "événementielles" faibles.
- Ajouter une loupe par carte pour poser une question contextuelle.
- Ne pas demander la clé API à chaque utilisation de la loupe (clé déjà intégrée).
- Permettre des runs manuels fréquents pour vérification.

4) Travaux réalisés (chronologie fonctionnelle)

4.1 Responsive mobile / paysage
- Ajout de règles CSS dédiées au paysage (max-height réduit).
- Activation du scroll sur carte active en paysage.
- Ajustement des hauteurs/paddings pour garder lisibilité et navigabilité.

4.2 Gestion du texte dans les cartes
- Ajustement de la logique de fit text (réduction taille police dans limites).
- Détection des cas de dépassement avec mode "expandable".
- Ouverture d'un overlay pour lire le contenu complet si nécessaire.

4.3 Qualité éditoriale des contenus
- Renforcement des prompts de génération pour mieux cibler les sujets pertinents.
- Correction récente : prompts assouplis et fiabilisés après un incident de digest IA vide.
- Nouvelle règle explicite de qualité de langue dans les prompts :
  - phrases complètes
  - articles/déterminants/connecteurs obligatoires
  - pas de style télégraphique
- Résumés raccourcis (moins de densité forcée) pour préserver clarté et tenue visuelle.

4.4 Fonction loupe (Q/R contextuelle)
- Ajout d'un bouton loupe sur chaque carte.
- Ajout d'un overlay de question/réponse.
- Passage de la clé API en mode injecté automatiquement (plus de saisie utilisateur).
- Amélioration des réponses :
  - suppression des artefacts du type [1], [2]
  - rendu structuré (titres/listes/gras si nécessaire)
  - style plus pédagogique
  - réponses courtes pour questions simples
- Correction UX importante :
  - header du panneau (dont bouton fermer) rendu sticky pour rester accessible même en réponse longue.

5) Problèmes majeurs rencontrés et corrections

5.1 "La loupe me redemande la clé API"
- Cause : versions servies pas toujours synchronisées / ancienne logique client.
- Correctif : clé injectée côté workflow/scripts dans HTML de gh-pages via placeholder base64.

5.2 Push GitHub bloqué par secret scanning (Push Protection)
- Cause : clé API détectée lors d'un push direct.
- Correctif : injection sous forme base64 et mise à jour via API GitHub Contents avec retry sur conflits (409).

5.3 Digest IA vide un matin
- Symptôme : today.json avec news: [].
- Cause : prompt trop strict sur filtres/priorités, pouvant mener à zéro résultat.
- Correctif :
  - prompt assoupli pour garantir 5 news
  - retry automatique (jusqu'à 3 tentatives) en cas de réponse insuffisante.

5.4 Email reçu "sans lancement manuel"
- Explication : un workflow_dispatch avait été déclenché pour déployer des corrections, et ce dispatch exécute aussi la chaîne complète incluant l'envoi email.
- État décidé : comportement conservé volontairement (email envoyé aussi sur run manuel) pour permettre les vérifications utilisateur.

6) État actuel (au moment du résumé)
- Multi-thèmes opérationnels.
- Workflow principal actif (schedule + manuel).
- Loupe disponible sur les cartes avec meilleure UX.
- Clé API non demandée à l'utilisateur dans le flux nominal.
- Qualité linguistique renforcée dans les prompts.
- Correction du risque "digest IA vide" déployée.

8) Contrainte produit à conserver
- Le contenu doit rester lisible dans le cadre des cartes sur mobile.
- En cas de texte long, l'UI doit proposer une lecture étendue sans casser la navigation.
- Les réponses loupe doivent être concises par défaut, détaillées seulement sur demande explicite de l'utilisateur.

9) Recommandation pour la suite
- Mettre en place des tests automatiques de validation JSON (champs non vides, counts attendus, longueur min/max).
- Ajouter un contrôle qualité linguistique post-génération (heuristiques simples) avant publication.
- Ajouter un mode "dry-run" du workflow pour tester déploiement sans impacter l'email si besoin futur.
